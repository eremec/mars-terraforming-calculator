# Терраформирование Марса — счётчик ресурсов

Мобильный веб-счётчик ресурсов для настольной игры «Терраформирование Марса» (базовая игра).
Каждый игрок ведёт свои ресурсы на своём телефоне; состояние хранится в `localStorage`.

**Приложение:** https://eremec.github.io/mars-terraforming-calculator/

## Установка на телефон (PWA)

Открой ссылку в браузере → «Поделиться» → «На экран Домой». Дальше работает как
обычное приложение, в том числе офлайн.

## Возможности

- TR и 6 ресурсов (производство и склад раздельно); у склада быстрые шаги `−10 −5 −1 / +1 +5 +10`.
- Кнопка «Производство» считает фазу по правилам базы (энергия→тепло, M€ = TR + произв.,
  остальное по производству), с превью и отменой.
- Конверсии 8 🌱 → озеленение и 8 🔥 → нагрев (обе +1 TR), счётчик поколений, сброс.

## Как работают обновления

PWA настроена так, чтобы обновления прилетали сами, без ручного сброса кэша:

- `index.html` отдаётся **network-first** — при наличии сети всегда загружается свежая версия,
  кэш используется только офлайн.
- Новый service worker активируется сразу (`skipWaiting` + `clients.claim`), а страница ловит
  смену контроллера и делает один тихий `reload` (состояние в `localStorage` не теряется).
- При каждом запуске и возврате фокуса вызывается `registration.update()`.

> **При деплое:** бампай версию кэша в `sw.js` (`const CACHE = 'tm-tracker-vN'`) — иначе браузер
> не увидит новый service worker и не подхватит обновление у уже установленного приложения.

## Разработка

Файл один — `index.html` (HTML + CSS + JS инлайном) плюс `manifest.webmanifest`, `sw.js` и иконки.
Локальный запуск:

```
python3 -m http.server 8000
# http://localhost:8000/
```

Иконки генерируются скриптом на Pillow (планета Марс с зелёными материками) — см. историю коммитов.
Деплой: `git push` в `main`, GitHub Pages пересобирает за ~минуту.
